outputStackTop

outputStackTop: number

Inherited from Frame.outputStackTop

  • Defined in packages/asm/libs/Frame.ts:271

Relative size of the output stack. The exact semantics of this field depends on the algorithm that is used.

When only the maximum stack size is computed, this field is the size of the output stack relatively to the top of the input stack.

When the stack map frames are completely computed, this field is the actual number of types in {@link #outputStack}.

TOP_IF_LONG_OR_DOUBLE: number = 8388608

Inherited from Frame.TOP_IF_LONG_OR_DOUBLE

  • Defined in packages/asm/libs/Frame.ts:79

Flag used for LOCAL and STACK types. Indicates that if this type happens to be a long or double type (during the computations of input frames), then it must be set to TOP because the second word of this value has been reused to store other data in the basic block. Hence the first word no longer stores a valid long or double value.
